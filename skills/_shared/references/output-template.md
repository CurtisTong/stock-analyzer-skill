# 统一输出规范（v2）

> 14 个 skill 共用，按需加载。`common/formatters.py` 提供程序化支持。
> 本规范是阅读侧契约；生成侧护栏详见
> [`./guardrails.md`](./guardrails.md)。

## 一、三段式结构（强制）

每个 skill 输出必须遵循以下三段渐进披露结构：

### 30 秒研判（必呈现，决策用）

```text
🎯 一句话研判(emoji + 倾向词,无"建议/买入/目标价")

[决策卡片六件套:方向/置信/仓位/价位/止损/最大风险]
[五层评级速览(可选)]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

- **首行**: `{emoji} {研判}` 例如 `🟢 偏多, 置信度 75%`
- **决策卡片**: 6 个字段必须同时出现
  - 倾向 / 置信度 / 参考仓位 / 现价 / 观察上沿 / 观察下沿
  - 最多加一条"风险收益比"
- **风险信号**: ⚠️ 最大风险一句话
- **合规提示**: 卡片下方 1 行斜体免责

### 风险与决策路径（必呈现）

```text
📌 核心矛盾 + 论点破灭触发器
```

- 核心矛盾 1-3 条(一句话一条)
- 论点破灭触发器(追踪哪些数据变化会推翻结论)
- **必须完整呈现** — 风险可见性是底线

### 详细论证（按章节跳读）

```text
[各 skill 自定章节]
- 数据 / 分析 / 推导
- 按股票类型路由(消费/科技/周期)
```

- 完整呈现论证章节,支持跳读
- 各 skill 按自身模板展开

### 数据护栏与免责（统一承载）

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🛡️ 数据护栏条
📅 数据时间戳
🔌 数据源 + 失败源
📜 免责声明
```

- 数据护栏条是合规与可审计的生死线
- 时间戳必须有,数据源标注失败源
- 免责声明是第三段冗余(共三段:卡片下斜体 + 护栏条内 + 页脚)

## 二、Emoji 白名单

| emoji | 含义 | 允许位置 | 频次上限 |
| --- | --- | --- | --- |
| 🎯 | 一句话研判标题 | 仅报告首行 | 1 次 |
| 🟢 | 看多 / 偏多 / 强势 | 仅关键判定处 | ≤3 次 |
| 🔴 | 看空 / 偏空 / 弱势 | 仅关键判定处 | ≤3 次 |
| 🟡 | 中性 / 观望 | 仅关键判定处 | ≤3 次 |
| ⚠️ | 数据缺失或硬风险 | 仅必要处 | ≤5 次 |
| 💡 | 专家视角提示 | 全文 | ≤3 次 |

**禁用**:
- ⛔ 任何"禁止"符号 → 改用 `**禁止**` 加粗或迁到护栏条
- 📊 二级标题前缀装饰 → 标题层级本身就是视觉锚点
- ⭐ 任何评分符号 → 改用字母评分
- 🔴 作为审查编号 → 移到护栏条

> 设计原则: emoji 是**路标**不是装饰,语义固定。

## 三、合规措辞

> A 股非持牌机构不得发布"投资建议"。
> 本工具非证券投资咨询业务持牌机构,任何"建议"动词触发合规红线。

**禁用措辞**(模板字符串中彻底删除):

| 禁用 | 替代 |
| --- | --- |
| 建议买入 / 卖出 / 加仓 / 减仓 | 研判 / 观察 / 倾向 |
| 目标价 XX 元(无依据时) | 观察上沿 / 隐含 PE |
| 止损位 / 止盈位 | 观察下沿 / 观察上沿 |
| 明日高开 / 今日尾盘 | (此话题不该出现) |
| 强烈推荐 / 强推 | 综合研判 / 置信度 X% |
| 明日涨跌 | (禁止) |

**合规措辞模板**:

```text
✅ "综合研判: 偏多, 置信度 80%"     ← 研判 + 数字
✅ "操作观察: +X.XX 突破 / -X.XX 跌破" ← 观察 + 触发价
✅ "多空倾向: 多头占优 4:3"           ← 表决
✅ "结论等级: 可参考 / 一般 / 不确定"  ← 强度词

❌ "**建议买入**"                ← "建议"踩线
❌ "**目标价 XX 元**"(无依据)    ← 监管红线
❌ "**明日高开**"                ← 隔日预判
```

**免责声明三处冗余**:

1. **决策卡片下方 1 行斜体**: 读完倾向立刻提醒
2. **数据护栏条内 1 行**: 报告末尾正式承担
3. **页脚完整段**: 监管认可的合规设计

## 四、章节处置原则

| 原则 | 含义 |
| --- | --- |
| 简洁 ≠ 删除 | 决策刚需保留并简化,不删除 |
| 路由 ≠ 普适 | 按股票类型条件渲染章节 |
| 删除 | 仅当与决策无关或监管已降级 |

**按股票类型路由**(详见 `guardrails.md` §五):
- 消费/医药 → 保留: 大盘 / 板块强度 / 北向偏好 / 分产品
- 科技/成长 → 保留: 大盘 / 行业 beta(一行)/ 流动性
- 周期(有色/化工/能源)→ 保留: 宏观桥 / 板块 / 情绪周期 / 产能
- 公用事业 → 仅保留: 大盘 / 板块强度

**移出个股报告的章节**:
- 组合相关性 → 移至 `/portfolio`
- IF/IC/IH 期货基差 → 移出(目标用户错位)
- 题材轮动 → 移出(长线投资者不需要)

**监管已降级,降级为附录/可选**:
- 北向资金(2024 年起监管限制披露)

## 五、评分体系(全文统一)

**主评分**:`A+/A/B+/B/C` 五档字母制(审查 #28)。
**辅助强度**: 🟢 强 / 🟡 中 / 🔴 弱。
**禁用**: ⭐ 任何评分符号。

| 维度 | 表达 | 示例 |
| --- | --- | --- |
| 五层评分 | 字母 + mini 进度条 | `基本面 A+ ████▌` |
| 强弱 | 进度条 | `强势 ████████░░` |
| 数字 | 加粗 + 千分位 | `**PE 18.4**` |
| 章节进度 | 横条 | `[1/7 决策] [2/7 市场] ...` |

## 六、视觉与可扫描性

- **每 3 张表格**强制插入 1 段"文字化图表"(sparkline / 进度条)
  作为视觉换气
- **首屏 ≤ 50 行**(`/stock quick` 模式)
- **深度报告 ≤ 280 行**(从原 475 行精简)
- 关键数字前置,详细论证后置
- 趋势/强度/占比一律用 `▁▃▅▇` 或 `████░░` 字符化表达
- 表格仅用于精确数值对比,不放主结论

## 七、quick 模式简化

quick 模式可省略详细内容,仅保留:
- 30 秒研判(决策卡片)
- 数据护栏与免责(护栏条 + 免责声明)

总行数 ≤ 50 行。

## 八、引用锚点

- **首行约定**:本规范首行为 🎯 一句话研判,与原文末行 `📊 数据时间戳`
  兼容(原文末行规范升级为页脚里的时间戳条目)
- **生成器护栏**: [`./guardrails.md`](./guardrails.md)
