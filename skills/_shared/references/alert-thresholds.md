# 预警与告警阈值

> `portfolio`（健康检查）和 `monitor`（实时推送）共用，按需加载。
> 修改此文件后，请同步在对应 skill 的 SKILL.md 里加"权威源指向本文件"。

## 持仓状态分级（portfolio 用）

| 状态    | 触发条件                                  | 操作建议     |
| ------- | ----------------------------------------- | ------------ |
| 🔴 破位 | 跌破关键支撑位                            | 减仓/止损    |
| 🟡 弱势 | 连续 2 日跑输板块                         | 观察/减仓    |
| 🟢 健康 | 在支撑位上方运行                          | 持有         |
| ⭐ 强势 | 板块领涨+放量                             | 持有/加仓    |

## 自选股状态分级（portfolio watchlist 用）

| 状态 | 触发条件 | 说明 |
| ---- | ---- | ---- |
| 🔴 已破止损 | 现价 ≤ target_sell | 止损价已跌破，策略失效，不应买入 |
| 🟡 接近止损 | 现价 > target_sell 且 (现价 - target_sell) / target_sell ≤ 3% | 接近止损，密切观察 |
| 🟡 接近买点 | 现价 > target_buy 且 (现价 - target_buy) / target_buy ≤ 5% | 接近目标买入价 |
| 🟢 到达买点 | 现价 ≤ target_buy | 到达目标买入价，可执行 |
| 🟢 观望 | 远离买卖点 | 继续等待 |

> **关键区分**：`现价 ≤ target_sell` 为"已破止损"（🔴），不是"接近"。"接近止损"仅在现价仍高于止损价但距止损 ≤3% 时使用。

## 推送触发阈值（monitor 用）

| 类型     | 触发条件         | 默认阈值       | 同源引用                              |
| -------- | ---------------- | -------------- | ------------------------------------- |
| 价格预警 | 涨跌幅超阈值     | ±3%            | 与持仓状态分级的"弱势/破位"互补       |
| 价格预警 | 触及支撑/压力位  | -              | 关键点位来自 `alert_engine.compute_key_levels` |
| 价格预警 | 涨跌停附近       | 距涨跌停 <1%   | A 股板块制度（主 10% / 创 20% / 北 30%） |
| 技术信号 | MACD 金叉/死叉   | -              | DIF 上穿/下穿 DEA                     |
| 技术信号 | 均线突破         | 20/60 日线     | -                                     |
| 技术信号 | 放量异动         | 量比 >2.0      | -                                     |
| 持仓风险 | 风险状态变更     | -              | 与上方"持仓状态分级"统一口径          |
| 持仓风险 | 连续跑输板块     | 2 天           | 与"🟡 弱势"一致                       |
| 市场环境 | 大盘涨跌幅       | ±2%            | 沪深300 / 上证指数                    |
| 市场环境 | 北向资金大幅流动 | ±50 亿         | 仅作参考；脚本若无北向数据则跳过      |

## 频率控制（monitor 强制）

- 同类消息 15 分钟去重
- 每日推送上限 20 条
- 非交易时段（15:05-09:25）静默
- 紧急预警（破位/止损）可 `urgent=True` 绕过每日上限（仍受去重窗口限制）

## Guardrails

- 阈值修改须同步更新 `monitor/SKILL.md` 和 `portfolio/SKILL.md` 的引用注释
- 跨平台推送（Bark/企微/钉钉）阈值一致；不同通道不做差异化阈值
- 不支持并发写入持仓数据，多进程场景需串行化
