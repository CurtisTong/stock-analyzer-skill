# Sector Analysis

板块深度分析——标的对比+多空博弈+配置建议。

## Usage

```text
/sector <板块名称> [模式: overview|compare|stock]
```

- `overview`（默认）：板块全景，核心标的+估值+走势+轮动位置
- `compare`：板块内标的横向对比，选出最优
- `stock`：板块内指定个股深度分析

示例：

- `/sector 机器人` — 机器人板块全景
- `/sector PCB compare` — PCB板块标的对比
- `/sector 医药 stock 恒瑞医药` — 医药板块内恒瑞分析

## Instructions

使用中文，重点突出对比和结论。先判断板块状态和首选标的，再解释数据依据。板块名称不在内置库时，先说明需要自行识别核心标的，不要编造代码。

输出遵循统一模板：首行为一句话结论，尾行为数据时间戳 + 数据源。详见 `../_shared/references/output-template.md`。

## Workflow Coordination

完整链路见包根目录 `workflow.md`。本 skill 位于 `market` 和 `screener`/`stock` 之间：

- 上游来自 `market`：接收强弱板块、市场风格和风险偏好。
- 下游到 `screener`：需要从板块里挑候选时，传递板块名称、核心标的池、推荐策略。
- 下游到 `stock`：用户指定板块内个股或需要深挖首选标的时，传递同业比较结论。
- 下游到 `portfolio`：用户问配置比例时，传递板块仓位上限、替代板块和退潮信号。

输出必须包含 `sector_view`（启动/主升/高潮/退潮/弱势）和首选/次选/回避名单。

### Step 1: 获取板块数据

根据板块名称，先查包根目录 `data/sector_stocks.json` 和 `data/sector_etf.csv`，再按 `../_shared/references/script-catalog.md` 获取核心标的行情+财务+K 线。脚本支持 `-j` JSON 排序。

### Step 2: 板块分析框架

**overview模式：**

| 分析维度 | 内容                    |
| -------- | ----------------------- |
| 板块定位 | 赛道空间+增长逻辑       |
| 核心标的 | 5-10只，PE/ROE/增速对比 |
| 估值水平 | 板块平均PE vs 历史分位  |
| 近期走势 | 5日/10日涨跌+量能       |
| 轮动位置 | 启动/主升/高潮/退潮     |

**compare模式：**

```
| 标的 | PE | ROE | 增速 | 毛利率 | 今日 | 评分 |
|------|-----|------|------|--------|------|------|
| 标的A | 14  | 25%  | +50% | 35%    | +2%  | ★★★★★ |
| 标的B | 45  | 8%   | +20% | 28%    | -1%  | ★★★   |
```

输出：最优标的+理由+买入区间

**stock模式：** 调用 `/stock` 的五层分析框架

如果用户要求板块内指定个股，直接复用 stock skill 的五层框架，但增加“与板块内其他核心标的比较”。

### Step 3: 配置建议

- 板块占组合建议比例
- 首选/次选/回避标的
- 买入时机+止损位

## Guardrails

- 板块结论必须区分“行业基本面”和“交易轮动”，不要把短线涨跌直接等同长期景气。
- 对无法确认的主题概念，明确标注“标的库未覆盖”并使用用户给出的标的。
- 不要给出确定性涨幅目标；用情景区间、触发条件和止损位表达。
