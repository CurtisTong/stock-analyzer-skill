# 深度分析报告模板（v2）

> 适用于 `/stock full` 与 `/stock debate` 输出。
> `/stock quick` 仅输出第一屏 + 第二屏 + 页脚，不展开第三屏。
>
> 前两屏承载**所有决策结论**，第三屏是**论证档案**，
> 用户通过章节编号锚点（`### 3.1 / 3.2 / ...`）跳读。
>
> 报告遵循 `../../_shared/references/output-template.md` v2 统一规范。
> 生成器护栏权威源：`../../_shared/references/guardrails.md`。

## 第一屏 · 研判（30 秒可读）

> 本节仅 1 个段落 + 1 个 KPI 行 + 1 个五维评分行 + 1 行风险提示。
> **不再有"决策卡片独立表格"或"五层速览独立 H3"**——信息密度归一。

```markdown
> 🟢 偏多 · 置信度 42% ▂▃▅▇▇ —— [一句话核心论点 + 时间窗口]
>
> - **核心矛盾**:[一句话张力,如"高增长退坡(+273%→+40%) vs 高估值(PE 60×)"]
> - **当前动作**:[一句话,如"🧐 观察 ¥282-285,不追高"]
>   _仓位 [X]% 为"观察仓"上限,非"建仓"建议_

**现价 18.42** | 上沿 22.10 (+20%) / 下沿 15.30 (-17%) | 风险收益比 **1 : 1.6**

基本面 A+ · 估值 A · 技术 B+ · 板块 A · 风控 B+

**⚠️ 最大风险**:[一句话触发条件,如"H1 预增低于中值 10% 即下调"]
_本卡片为量化倾向研判，不构成投资建议。数据护栏见页脚。_
```

**字段映射说明**:
- 倾向 emoji: 🟢 偏多 / 🔴 偏空 / 🟡 中性
- 置信度: 整数百分比 + 进度字符(`▁▃▅▇` 五档: ≤20/40/60/80/>80)
- 一句话核心论点: 含时间窗口(如"半年内" / "Q3 财报披露前")
- **核心矛盾**:1 行,只用最锋利的张力表达(如"高增长退坡 vs 高估值")
- **当前动作**:1 行,必须是动词开头的具体行为(如"观察 ¥282" / "减仓 60%→40%")
- **仓位警示**:在"当前动作"下用斜体标注"观察仓/建仓"语义边界,防止激进偏差
- 五维评分: 字母制 `A+/A/B+/B/C`，按顺序输出基本面/估值/技术/板块/风控
- 最大风险: 1 行，必须含可量化触发条件

## 页脚 · 数据护栏（统一承载）

> 本节承担 **数据真实性 / 风险分级 / 免责 三重职责**。分级采用三色:
> - 🔴 **高级风险**(红色):核心结论存疑,影响仓位/止损决策
> - 🟠 **中级局限**(橙色):模型/数据有结构性局限,影响边际判断
> - 🔵 **低级瑕疵**(蓝色):字段缺失或口径不严谨,不影响主结论
>
> 三色分级仅在页脚承载,不进第一屏(避免视觉过载回流)。
> 第一屏的"⚠️ 最大风险"承担 1 行精炼提示;详细分级见本节。

## 使用说明

- 仅在 `/stock full` 或 `/stock debate` 模式加载本模板。

- `[...]` 占位符必须替换为真实数据；禁止保留占位符。

- 模板字符串内的 `scripts/xxx.py <代码>` 即为可复现的执行命令。
- emoji 必须使用白名单内（详见 `output-template.md` §二）。
- 措辞必须走合规替换（详见 `output-template.md` §三）。
- 生成器护栏必须遵守（详见 `guardrails.md`）。
