## P1-A: beichi.py 坐标系错位（趋势/盘整背驰完全失效）

**根因**: `chan_zhongshu` 中 `xd_start`/`xd_end` 是 **xd_list 索引**（0, 1, 2...），但 `beichi.py` 将它们当作 **merged-bar 索引** 与 `bi["start_idx"]`/`bi["end_idx"]` 比较。结果：
- `entry_bi`（进入段）几乎总是 `None`（因为 `bi["end_idx"] <= 0` 几乎不成立）
- `exit_bi`（离开段）总是选到 `bi_list[0]`（因为 `bi["start_idx"] >= 2` 对几乎所有 bi 成立）
- 趋势背驰和盘整背驰检测在真实数据上**完全失效**
- 下游 `maidian.py` 的一买/一卖信号因此降级为弱信号

**修复方案**: 在 `chan_zhongshu` 构建中枢时存储 `bi_start`/`bi_end`（bi_list 索引），`beichi.py` 改用 bi_list 索引直接定位进入/离开笔。

### 文件修改

**1. `scripts/chan/zhongshu.py`** — 中枢字典新增 `bi_start`/`bi_end` 字段
- 初始构建（~line 29-38）: `"bi_start": xd_list[i]["start_bi"]`, `"bi_end": xd_list[i+2]["end_bi"]`
- 合并步骤（~line 54-61）: `"bi_start": last["bi_start"]`, `"bi_end": zs["bi_end"]`

**2. `scripts/chan/beichi.py`** — 改用 bi_list 索引定位进入/离开笔
- 趋势背驰（~line 47-62）:
  ```python
  bi_start = last_zs.get("bi_start", 0)
  bi_end = last_zs.get("bi_end", len(bi_list) - 1)
  entry_bi = bi_list[bi_start - 1] if bi_start > 0 else None
  exit_bi = bi_list[bi_end + 1] if bi_end + 1 < len(bi_list) else None
  ```
- 盘整背驰循环（~line 98-113）: 同样改用 `bi_start`/`bi_end`
- 保留 `xd_start`/`xd_end` 字段不变（向后兼容）

**3. `tests/test_chan.py`** — 新增真实场景背驰测试
- `_make_zs` 辅助函数: 添加 `bi_start`/`bi_end` 参数
- 新增 `TestChanBeichiWithZhongshu` 测试类: 用非空 `zs_list` 测试背驰，验证 entry/exit 笔正确选取

## P1-B: xianduan.py 特征序列实现与文档/缠论标准矛盾

**问题**: 
1. 文档说"除起点笔外的所有笔"，代码用 `seg_bis[:-1]`（排除终点笔，方向反了）
2. 缠论标准：上升段的特征序列 = **下跌笔的低点**，代码取所有笔的低点（混入了上涨笔的低点）

**修复方案**: 按方向过滤特征序列笔，修正排除逻辑（排除起点笔 = `seg_bis[1:]`），修正 docstring。

### 文件修改

**`scripts/chan/xianduan.py`** — `_feature_sequence_break` 修正
- 上升段: `feature_lows = [b["low"] for b in seg_bis[1:] if b["direction"] == "down"]`
- 下降段: `feature_highs = [b["high"] for b in seg_bis[1:] if b["direction"] == "up"]`
- 空 list 守卫: 无特征元素时 `return False`
- docstring 修正：明确"特征序列 = 除起点笔外的反向笔"

## P1-C: classifier.py "航空"关键词碰撞（航空公司误判为军工）

**问题**: "航空"同时出现在军工（line 370）和交运（line 373）关键词列表。军工先检查，导致"南方航空"、"中国国航"等被误判为军工。
