## 阶段 A：P0 修复（8 项）

### A1. 缠论 merge.py 首对包含方向默认值错误
**文件**: `scripts/chan/merge.py:32`
**问题**: `direction = "up"` 硬编码。若首对K线包含且实际趋势向下，合并取"涨势取高高"而非"跌势取低低"，下游所有计算错误。
**修复**: 根据 `bars[0]` vs `bars[1]` 的 close 比较初始化方向：
```python
direction = "up" if len(bars) < 2 or bars[1].get("close", 0) >= bars[0].get("close", 0) else "down"
```

### A2. MA 均线交叉索引对齐错误
**文件**: `scripts/strategies/patterns/ma_volume_strategy.py:71`
**问题**: `idx_short = i + offset_short - offset_long + offset_long` 简化为 `i + offset_short`，但正确对齐应为 `idx_short = i + offset_short` ... 实际上简化后就是 `i + offset_short`，但 MA 系列对齐应该是 `idx_short = i`（两个 MA 系列都是尾部对齐 closes，遍历 ma_long 时 idx_short 应该与 i 同步偏移）。让我重新分析：
- `ma_short_series[k]` 对应 `closes[k + offset_short]`
- `ma_long_series[i]` 对应 `closes[i + offset_long]`
- 要比较同一天的短/长 MA，需 `k + offset_short == i + offset_long` → `k = i + offset_long - offset_short`
- 当前代码 `idx_short = i + offset_short`，偏差 `2*(offset_long - offset_short)`
**修复**: `idx_short = i + offset_long - offset_short`

### A3. 缠论三买/三卖条件过于宽松
**文件**: `scripts/chan/maidian.py:84,143`
**问题**: 三买检查 `recent_low > last_zs["zd"]`（回踩低点高于ZD），缠论标准应高于ZG。三卖对称：`recent_high < last_zs["zg"]` 应改为 `< last_zs["zd"]`。
**修复**: 84 行 `recent_low > last_zs["zg"]`；143 行 `recent_high < last_zs["zd"]`。

### A4. screening_pipeline 排序后 zip 错位
**文件**: `scripts/business/screening_pipeline.py:246-248`
**问题**: `rows_p1.sort()` 重排序后 `zip(quotes, rows_p1)` 配对错误——stock A 的 quote 搭配 stock B 的 score。
**修复**: 先配对再排序：
```python
pairs = list(zip(quotes, rows_p1))
pairs.sort(key=lambda qr: qr[1].get("score", 0), reverse=True)
top_quotes = [q for q, r in pairs if r.get("score", 0) > 0][:top_n_phase2]
```

### A5. long_term.py 增长维度字段名错误（永远为0）
**文件**: `scripts/business/long_term.py:184-185,200-201`
**问题**: `finance.get("revenue_growth")` 和 `finance.get("profit_growth")` 不存在于 FinanceRecord，正确名为 `revenue_yoy` / `net_profit_yoy`。导致每只股票增长维度默认扣 30 分。
**修复**: 改为 `finance.get("revenue_yoy")` / `finance.get("net_profit_yoy")`，删除死回退字段 `YYZSRTBZZ`/`JLRTBZZ`。

### A6. risk_metrics VaR/CVaR abs() 对正收益返回正风险值
**文件**: `scripts/business/risk_metrics.py:37,51`
**问题**: `abs(sorted_r[idx])` 对全正收益仍返回正 VaR，夸大风险。
**修复**: 改为 `max(0.0, -sorted_r[idx])` / `max(0.0, -sum(tail)/len(tail))`。

### A7. 监控 resistance_touch 无上限（持续误报）
**文件**: `scripts/monitor/rules.py:106`
**问题**: `price >= lv * 0.99` 无上界，价格远超压力位后每次检查都触发。support_touch 正确用了 `lv*0.99 <= price <= lv*1.01`。
**修复**: `lv * 0.99 <= price <= lv * 1.01`（与支撑位一致）。

### A8. Portfolio manager 代码大小写不一致
**文件**: `scripts/portfolio/manager.py:279,332,387,444,467,490,523,561`
**问题**: `_find_position` 用 `.lower()` 不敏感匹配，但 `_apply` 闭包用 `p["code"] == code`（区分大小写），mixed-case 代码导致操作静默失败。
**修复**: 所有 `_apply` 闭包的 `p["code"] == code` 改为 `p["code"].lower() == code.lower()`，或确保 `_load` 时统一 normalize 为小写。
