# ADR-003: 专家投票系统（Expert Panel）

| 字段     | 值                          |
| -------- | --------------------------- |
| 状态     | Accepted                    |
| 日期     | 2026-02-10                  |
| 决策者   | curtis                      |
| 相关模块 | `experts/`, `scripts/business/stock_analysis.py` |

## 背景

单一评分模型无法兼顾不同投资风格（价值/成长/动量/技术）。
巴菲特看好的股票可能被动量交易者看空，二者都有道理但视角不同。

## 决策

采用**多专家投票**架构：

- 16 份专家人设（8 active = 5 长线 + 3 短线），每份独立评分 0-100
- 分为 `long_term` / `short_term` 两组，分别投票
- `decide()` 聚合：组内加权平均 → 组间权重 → 最终多空判定
- ST 股票触发**硬否决**（momentum_trader 直接返回低分）

```
专家评分(0-100) ─> 组内加权 ─> 组间投票 ─> 多空结论
     ↑                                          ↓
  各专家独立判断 ──────────> 推理链 + 置信度
```

## 理由

1. **视角多元**：不同投资风格各有适用场景，避免单一模型偏见
2. **可解释性**：每份专家的推理链保留在结果中，用户可查看分歧
3. **可扩展**：新增专家只需注册 YAML + 实现 `score()`，不改聚合逻辑
4. **ST 防护**：硬否决确保 ST/退市风险股不被动量策略误推

## 后果

- 正面：评分结果附带推理链和分歧度，透明可审计
- 正面：专家可独立增减，不影响其他专家
- 负面：16 份人设需维护一致性（YAML ↔ registry ↔ md 三源同步）
- 缓解：`test_yaml_consistency.py` 自动校验三源对齐
