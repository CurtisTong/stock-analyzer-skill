# ADR-004: 回测偏差防护（Backtest Bias Prevention）

| 字段     | 值                              |
| -------- | ------------------------------- |
| 状态     | Accepted                        |
| 日期     | 2026-04-05                      |
| 决策者   | curtis                          |
| 相关模块 | `scripts/backtest/`, `scripts/strategies/patterns/` |

## 背景

回测系统容易产生"过拟合幻觉"：在历史数据上表现优异的策略，
实盘可能大幅亏损。常见偏差来源包括前视偏差（lookahead bias）、
幸存者偏差、过度参数优化等。

## 决策

在回测和策略系统中实施多层偏差防护：

### 1. 前视偏差标注
所有使用未来数据的计算点（如 KDJ 收敛、均线对齐）
在代码中标注 `# P1-XX: 前视检查` 或 `# 注意：此处使用未来数据`，
便于审计和 CI 扫描。

### 2. 样本内/外分离
`patterns/config.json` 每个策略条目包含：
- `backtest_results`：样本内拟合结果（明确标注 disclosure）
- `multi_stock_validation`：多股交叉验证
- `oos_validated`：机器可读的样本外验证标志（当前均为 `false`）

### 3. 回测可视化标注
`backtest/visualize.py` 在报告中明确标注：
- regime overlay 状态（是否应用了市场环境调整）
- 样本内/外区间
- 策略参数来源（拟合 vs 验证）

### 4. 因子共线性诊断
`compute_factor_correlation_matrix()` 输出 9 因子的 Pearson 相关矩阵，
供诊断因子冗余（不改变打分，仅诊断）。

## 理由

1. **诚实标注**：不隐藏回测的局限性，`oos_validated: false` 明确告知未经验证
2. **可审计**：前视偏差标注让审查者快速定位风险点
3. **机器可读**：`oos_validated` 字段供下游过滤（如只选已验证策略）

## 后果

- 正面：回测结果可信度提升，用户知晓样本内拟合风险
- 正面：CI 可扫描前视标注覆盖率
- 负面：所有策略当前 `oos_validated: false`，需未来补充 walk-forward 验证
