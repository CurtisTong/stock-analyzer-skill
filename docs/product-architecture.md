## 一、产品定位

### 1.1 产品概述

**stock-analyzer-skill** 是一款面向 A 股市场的智能股票分析工具，基于 Claude Code 平台运行。通过自然语言交互，为投资者提供基本面分析、技术面分析、选股筛选、仓位管理等全维度投资辅助。

### 1.2 核心价值

| 价值主张   | 描述                                    |
| ---------- | --------------------------------------- |
| **智能化** | AI 驱动的多因子选股，减少主观判断       |
| **实时性** | 多数据源自动切换，秒级响应行情          |
| **专业性** | 融合缠论、16 人专家圆桌等机构级分析方法 |
| **零门槛** | 无需编程，通过自然语言即可使用          |
| **零依赖** | 核心脚本纯 stdlib，无需安装额外库       |
| **零配置** | 内置预置股票池，无需 token 即可使用     |

### 1.3 目标用户

| 用户类型       | 需求场景                     |
| -------------- | ---------------------------- |
| **个人投资者** | 选股参考、仓位管理、风险提示 |
| **量化爱好者** | 策略回测、因子研究           |
| **理财师**     | 客户持仓分析、投资建议生成   |

## 二、技术架构

### 2.1 三层架构设计

```
┌─────────────────────────────────────────────────────────────────┐
│                     stock-analyzer-skill                        │
├─────────────────────────────────────────────────────────────────┤
│  📡 接口层 (13 Skills)                                           │
│  ├── .claude/skills/ - Claude Code skill 源                    │
│  └── .agents/skills/ - Codex workspace skill 源                │
├─────────────────────────────────────────────────────────────────┤
│  🧠 业务层 (scripts/business/)                                  │
│  ├── stock_analysis.py - 个股分析服务                           │
│  └── screening_service.py - 选股服务                            │
├─────────────────────────────────────────────────────────────────┤
│  📊 数据层 (scripts/)                                            │
│  ├── fetchers/ - 多数据源适配器（27 个数据源 × 7 数据域 = 35 类）│
│  ├── common/ - 基础设施 (HTTP/缓存/异常/熔断器)                 │
│  ├── config/ - YAML 配置外部化                                  │
│  ├── strategies/ - 选股策略系统                                 │
│  ├── technical/ - 技术分析模块                                  │
│  └── portfolio/ - 持仓管理模块                                  │
└─────────────────────────────────────────────────────────────────┘

> **数据源数量核对口径**：27 个真实数据源 fetcher（quote 10 + kline 9 + finance 2 + flow 2 + lhb 1 + event 2 + chip 1），按 7 个数据域覆盖 35 类查询类型。`scripts/fetchers/_common/` 下另有 3 个 helper 文件（`tushare_check` / `pytdx_pool` / `pytdx_meta`）属于基础设施而非数据源。

### 2.2 核心技术组件

| 组件                   | 位置                             | 功能                   |
| ---------------------- | -------------------------------- | ---------------------- |
| **BaseFetcher**        | `scripts/common/__init__.py`     | 数据源抽象基类         |
| **CircuitBreaker**     | `scripts/common/__init__.py`     | 线程安全熔断器（三态） |
| **DataFetcherManager** | `scripts/common/__init__.py`     | 多源故障转移管理器     |
| **ConfigLoader**       | `scripts/config/loader.py`       | YAML 配置加载器        |
| **StrategyRegistry**   | `scripts/strategies/registry.py` | 选股策略注册中心       |

### 2.3 数据源矩阵

> **统计口径**：fetcher 数 = 真实数据源 Python 文件数（`scripts/fetchers/` 下，不含 `_common/` helper）；查询类型数 = 该数据域覆盖的不同查询维度。

| 数据类型     | fetcher 数 | 查询类型数 | 主要来源                                                                              |
| ------------ | ---------- | ---------- | ------------------------------------------------------------------------------------- |
| **行情数据** | 10         | 1          | 腾讯、东财、新浪、雪球、同花顺、efinance、akshare、tushare、pytdx、yfinance          |
| **K线数据**  | 9          | 5+         | 新浪、东财、腾讯、efinance、akshare、baostock、pytdx、tushare、yfinance               |
| **财务数据** | 2          | 3+         | 东财、akshare                                                                          |
| **资金面**   | 1          | 3          | 东财（融资融券 / 股东户数 / 十大流通股东）                                            |
| **资金流向** | 2          | 2          | 东财（北向资金 / 个股资金流）、新浪                                                    |
| **龙虎榜**   | 1          | 2          | 东财（明细 / 席位）                                                                    |
| **事件日历** | 1          | 3          | 东财（财报披露 / 限售解禁 / 分红送转）                                                |
| **合计**     | **26**     | **33**     | 7 个数据域（quote / kline / finance / chip / flow / lhb / event）                     |

| -------- | ----------------- | -------- | ------- | ---------- | ---------- |
| 金融     | 银行/保险/证券    | 10%      | 8       | -          | 90%        |
| 消费     | 食品饮料/纺织服装 | 15%      | 25      | 35%        | 60%        |
| 科技     | 电子/计算机/通信  | 12%      | 40      | 25%        | 70%        |
| 医药     | 医疗器械/生物制药 | 15%      | 30      | 40%        | 65%        |
| 制造     | 机械设备/汽车     | 12%      | 20      | 20%        | 70%        |
| 能源     | 煤炭/石油/化工    | 10%      | 15      | 15%        | 75%        |
| 地产     | 房地产开发        | 8%       | 10      | 20%        | 80%        |
| 周期     | 钢铁/有色/建材    | 10%      | 15      | 15%        | 75%        |

### 2.5 选股策略系统（五因子模型）

5 种内置策略（`scripts/strategies/registry.py`）：

| 策略            | 市场环境      | 质量 | 估值 | 动量 | 流动性 | 波动率 |
| --------------- | ------------- | ---- | ---- | ---- | ------ | ------ |
| balanced        | 震荡/方向不明 | 25%  | 20%  | 20%  | 15%    | 20%    |
| quality_value   | 价值修复/防守 | 35%  | 30%  | 5%   | 12%    | 18%    |
| growth_momentum | 进攻行情/主线 | 18%  | 15%  | 35%  | 12%    | 20%    |
| defensive       | 缩量弱市/避险 | 25%  | 22%  | 8%   | 12%    | 33%    |
| turning_point   | 超跌修复/拐点 | 18%  | 18%  | 32%  | 14%    | 18%    |

### 2.6 五因子详解

| 因子       | 文件位置                           | 评分逻辑                                                    |
| ---------- | ---------------------------------- | ----------------------------------------------------------- |
| **质量**   | `strategies/factors/quality.py`    | ROE趋势、净利增速、营收增速、毛利率、负债率、经营现金流/EPS |
| **估值**   | `strategies/factors/valuation.py`  | PE、PB、PEG（市销��），支持亏损股评估                       |
| **动量**   | `strategies/factors/momentum.py`   | 20日收益、趋势、量能比、MACD金叉/死叉、RSI                  |
| **流动性** | `strategies/factors/liquidity.py`  | 成交额、总市值、换手率（板块差异化）                        |
| **波动率** | `strategies/factors/volatility.py` | 历史收益率标准差，低波动得高分（A股低波动异象）             |

## 四、技术特性

### 4.1 线程安全

- CircuitBreaker 使用 `threading.Lock` 保护状态
- fetchers 延迟加载使用双重检查锁定
- 并发执行使用 ThreadPoolExecutor

### 4.2 多数据源故障转移

- 7 个行情源、8 个 K 线源、3 个财务源
- 按优先级自动切换（BaseFetcher.priority）
- 缓存降级机制：主数据源失败时返回缓存数据

### 4.3 行业差异化

- 30+ 行业阈值配置（`data/industry_thresholds.json`）
- 板块流动性差异化（主板/创业板/科创板/北交所）
- 避免系统性误判

### 4.4 可观测性

- 健康检查命令：`python3 scripts/monitor/health.py`
- 数据源状态实时监控
- 缓存统计信息

### 4.5 风控体系

- 集中度控制：单只≤15%、单行业≤30%
- 时间止损：20日无正收益减半仓，40日清仓
- 极端预案：大盘单日跌幅>5%减至30%
- 硬过滤：ST/退市/商誉/质押/涨跌停

### 5.2 投资决策流程

```
1. 输入股票代码
       ↓
2. 数据获取 (多源自动切换)
       ↓
3. 五层分析 (基本面→估值→技术→板块→风险)

5. 仓位建议 (凯利公式 + 阈值控制)

## 七、竞争优势
