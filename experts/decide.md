# 专家圆桌决策引擎

> 版本：v2.2.0（2026-06-18）
> 更新：v2.1.0 合并型专家 + v2.2.0 动量派补齐趋势跟踪盲区

> 整合 9 位 active 专家的独立评分，生成加权投票结果。用于 stock skill debate 模式。

## 一、投票机制

### 1.1 评分与方向判定

每位专家按自身评分矩阵（见各专家 §九）对目标股票打分 0-100：

| 分数区间 | 方向     | 含义                       |
| -------- | -------- | -------------------------- |
| ≥70      | 强烈看多 | 该专家框架下明确买入信号   |
| 60-69    | 看多     | 有条件看多，需确认其他维度 |
| 40-59    | 中性     | 无明确方向，观望           |
| 30-39    | 看空     | 该专家框架下存在显著风险   |
| <30      | 强烈看空 | 触发一票否决或多项硬伤     |

### 1.2 分组计票

```
长线组 = 巴菲特 + 林奇 + 索罗斯 + 段永平 + 价值双锚 + 行业专家 + 机构派 + 风险管理（v2.1.0 扩展 + 补盲区）
短线组 = 徐翔 + 赵老哥 + 养家 + 作手新一 + 题材龙头 + 情绪技术 + 动量派（v2.1.0 合并 + v2.2.0 动量派）

长线组平均分 = Σ(长线专家分数) / 长线专家数
短线组平均分 = Σ(短线专家分数) / 短线专家数
```

> v2.2.0 起，长线 active = 6（lynch + soros + value_anchor + sector_specialist + institution + risk_manager），短线 active = 3（topic_leader + emotion_tech + momentum_trader）。v2.1.0 时短线 active 仅为 2（topic_leader + emotion_tech），v2.2.0 新增动量派后扩为 3。传统 8 人圆桌模式通过 `list_legacy_experts()` 显式调用。

**投资期限权重**（基础综合分由投资期限决定）：

| 投资期限          | 长线权重 | 短线权重 | 说明                                           |
| ----------------- | -------- | -------- | ---------------------------------------------- |
| 短期操作（<1月）  | 35%      | 65%      | 短线为主，但注册制后日内波动下降，短线窗口拉长 |
| 中期持有（1-6月） | 40%      | 60%      | 量化主导结构市，趋势持续性增强，偏短线参照     |
| 长期投资（>6月）  | 70%      | 30%      | 以长线专家为主                                 |

```
基础综合分 = 长线组平均分 × 长线权重 + 短线组平均分 × 短线权重
```

### 1.3 多空计数

- 长线看多票 = 长线组中分数≥60的人数
- 长线看空票 = 长线组中分数≤39的人数
- 短线看多票 = 短线组中分数≥60的人数
- 短线看空票 = 短线组中分数≤39的人数

## 二、市场环境 → 权重动态调整

运行 debate 前，先获取大盘数据判断市场状态，按以下映射调整长线/短线权重：

| 市场状态          | 判断条件（增加宽度指标）                                    | 长线权重 | 短线权重 | 主导专家                   |
| ----------------- | ----------------------------------------------------------- | -------- | -------- | -------------------------- |
| **牛市/进攻**     | 沪深300 > MA20 + 成交放量 + 上涨家数比>60% + 新高新低比>1.5 | 40%      | 60%      | 林奇、动量派、题材龙头     |
| **熊市/防守**     | 沪深300 < MA20 + 缩量 + 上涨家数比<40% + 新高新低比<0.5     | 60%      | 40%      | 价值双锚、动量派、情绪技术 |
| **防御型**        | 沪深300 < MA20 + 量能偏弱 + 上涨家数比30-45%（结构性分化）  | 65%      | 35%      | 价值双锚、风险管理、机构派 |
| **震荡**          | 沪深300在MA20±3% + 上涨家数比40-60% + 板块分化              | 55%      | 45%      | 价值双锚、情绪技术、动量派 |
| **极度恐慌/冰点** | 上涨家数比<20%连续2日 + 跌停>50家 + 新高新低比<0.2          | 60%      | 40%      | 段永平、情绪技术、索罗斯   |
| **极度亢奋/泡沫** | PE>历史90%分位 + 上涨家数比>75% + 两融余额占比>10%          | 70%      | 30%      | 价值双锚、风险管理         |

> **v2.2.0 起**：动量派（momentum_trader）作为趋势跟踪专家在牛市/震荡/熊市初期均高度相关，与短线题材龙头形成"动量+事件"双视角。`stock debate` 默认调用 active 9 人时，动量派纳入短线组合票。

调整后综合分 = 长线组平均分 × 调整后长线权重 + 短线组平均分 × 调整后短线权重

## 三、冲突解决规则

> v2.2.0 适配：长线组 6 人、短线组 3 人。统一 67% 多数阈值。

| 情况       | 长线组(6人) | 短线组(3人) | 裁决          | 仓位调整           |
| ---------- | ----------- | ----------- | ------------- | ------------------ |
| 双一致看多 | ≥4/6看多    | ≥2/3看多    | **强烈看多**  | 标准仓位           |
| 双一致看空 | ≥4/6看空    | ≥2/3看空    | **强烈看空**  | 不建仓，持有则清仓 |
| 长线主导多 | ≥4/6看多    | 分歧        | **看多**      | 标准仓位×0.8       |
| 长线主导空 | ≥4/6看空    | 分歧        | **看空**      | 不建仓             |
| 短线主导多 | 分歧        | ≥2/3看多    | **谨慎看多**  | 标准仓位×0.5       |
| 短线主导空 | 分歧        | ≥2/3看空    | **谨慎看空**  | 减仓50%            |
| 全面分歧   | 分歧        | 分歧        | **中性/观望** | 不操作             |
| 巴菲特否决 | 巴菲特≤39   | 任意        | **看空**      | 巴菲特看空×1.5权重 |

**特殊规则**：

- 巴菲特评分≤39（看空）→ 中长期投资（≥1月）模式下触发否决权，最终方向至少降一级；短期操作（<1月）模式下不触发否决权，但长线组平均分额外×0.8 降权
- 养家评分<30 且非冰点期 → 短线组所有评分额外×0.7（仅评分降权，不影响仓位上限）
- 养家判定冰点期且评分<30 → 不降权其他专家，标注"冰点机会，需确认转折信号"（冰点=下一轮起爆点，与降权逻辑矛盾）
- 任一专家触发一票否决 → 该专家评分为0，且标注否决原因

## 五、使用流程（stock skill debate 模式）

1. **获取数据**：使用 `scripts/data/` 获取目标股票完整数据
2. **获取大盘数据**：使用 `get_quote("sh000001")` 或 `get_quote("sh510300")` 判断市场环境
3. **专家打分**：每位专家按各自 `§九 评分矩阵` 对目标股票独立打分（使用步骤1的数据）
4. **应用调整**：根据市场环境（步骤2）调整长线/短线权重
5. **冲突解决**：按 §三 规则处理分歧，特别注意巴菲特否决权和养家情绪周期
6. **输出结果**：按 §四 格式输出，含评分表、方向、风险、仓位

## 六、信心指数计算

信心指数考虑三个维度：专家一致性、综合分绝对值和历史校准偏差。

### 6.1 基础一致性

```
均值 = 9 位 active 专家评分的均值
CV = (9 位 active 专家评分的标准差 / 均值) if 均值 > 0 else 1  # 变异系数，越小越一致
一致性分 = 100 - (CV × 150)  # CV=0→100分，CV≈0.33→50分
capped = max(0, min(100, 一致性分))
```

### 6.2 历史校准偏差（2026新增）

跟踪每位专家的历史预测准确率，动态调整其在信心指数中的贡献权重。

```
# 每位专家维护 event_count / correct_count
校准率 = correct_count / event_count  （若无历史数据则取 0.5）

# 校准偏度：9 位 active 专家校准率的离散度
校准均值 = Σ(校准率) / 8
校准CV = 标准差 / 校准均值（校准均值>0时）

# 整体可执行性修正
校准因子 = 校准均值 × (1 - min(校准CV, 0.5))
校准调整 = (校准因子 - 0.5) × 2  # 归一化到 [-1, 1]
```

### 6.3 最终信心指数

```
基础信心 = (capped × 0.45 + 综合分 × 0.45 + 校准调整 × 10 × 0.1)
信心指数 = max(0, min(100, 基础信心))
```

校准调整的权重 0.1 确保历史偏差贡献不超过 ±10 分。

信心指数阈值：

- ≥75：高度可信，可执行
- 60-74：可信，但需额外确认
- 40-59：分歧较大，仅试探性仓位
- <40：不可靠，建议观望

### 6.4 校准数据存储

校准数据持久化在 `data/expert_calibration.json`，格式：

```json
{
  "buffett": {"events": 50, "correct": 32, "last_updated": "2026-06-01"},
  "lynch": {"events": 45, "correct": 27, "last_updated": "2026-06-01"},
  ...
}
```

## 七、单组模式（长线-only / 短线-only）

当用户指定 `debate 长线` 或 `debate 短线` 时，仅运行对应组4位专家，不涉及跨组加权。

### 7.1 组内投票规则

| 组内状态              | 裁决         | 仓位建议     |
| --------------------- | ------------ | ------------ |
| ≥3/4 看多             | **看多**     | 标准仓位     |
| 3/4 看多 + 1 一票否决 | **看多**     | 标准仓位×0.7 |
| 2:2 分歧              | **中性**     | 不操作       |
| ≥3/4 看空             | **看空**     | 不建仓/减仓  |
| 全票强烈看多          | **强烈看多** | 标准仓位×1.2 |
| 全票强烈看空          | **强烈看空** | 清仓         |

### 7.2 特殊规则（仅短线模式）

- 养家判定为"退潮期/冰点" → 其余3位短线专家评分×0.7
- 养家一票否决（退潮期）生效 → 短线组整体降为"中性/观望"

### 7.3 信心指数（单组模式）

```
均值 = 4人评分的均值
CV = (4人评分的标准差 / 均值) if 均值 > 0 else 1
一致性分 = 100 - (CV × 150)
capped = max(0, min(100, 一致性分))
信心指数 = capped × 0.35 + 组平均分 × 0.65
```

阈值同 §六（≥75 可执行，60-74 可信，40-59 试探，<40 观望）。

### 7.4 输出格式（单组模式）

简化版，仅显示该组4位专家：

```text
## 专家圆桌投票结果（长线模式 / 短线模式）

| [专家3] | XX | 看多/空 | [1句话] |
| [专家4] | XX | 看多/空 | [1句话] |

## 组内汇总
- 平均分: XX/100 | 看多X票 / 看空X票
- **最终方向: [看多/中性/看空]**
- 信心指数: XX/100

## 风险提示
[列出看空专家的核心担忧]

- 推荐仓位: X%
```
